## Finding

1. `apps/web/src/features/workspace/components/WorkspaceSurface.tsx`
   `commitDocumentRename()` now treats the center-pane title field as a filename editor and forwards the raw input directly to `localWorkspaceFileClient.renameFile(...)`.

   Why this is a bug:
   - The new UI displays and edits the filename stem only.
   - The backend/browser rename sanitizers still accept a user-supplied extension and will preserve that new extension when present.
   - A user can type `meeting-notes.txt` into the title field for `meeting-notes.md`, and the file will be renamed to `.txt` instead of keeping the current extension.

   Affected paths:
   - `apps/web/src/features/workspace/components/WorkspaceSurface.tsx`
   - `apps/web/src/features/workspace/data/localWorkspaceFileClient.ts`
   - `apps/desktop/src/main/localWorkspaceService.ts`

   Recommended fix:
   - Strip any user-supplied extension before calling the rename API from the inline title editor, or add a dedicated rename-by-title path that always preserves the current extension.
